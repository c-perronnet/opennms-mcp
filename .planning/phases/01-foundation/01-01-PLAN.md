  <action>
Create `package.json` at project root with these exact fields:

```json
{
  "name": "opennms-mcp",
  "version": "1.0.0",
  "description": "MCP server for OpenNMS — lets Claude read and act on alarms, nodes, events, assets, and categories via natural language",
  "type": "module",
  "main": "dist/index.js",
  "scripts": {
    "build": "tsc",
    "start": "node dist/index.js",
    "dev": "tsc --watch"
  },
  "engines": {
    "node": ">=18"
  },
  "dependencies": {
    "@modelcontextprotocol/sdk": "^1.27.1",
    "axios": "^1.13.6",
    "zod": "^3.25.0"
  },
  "devDependencies": {
    "typescript": "^5.9.3",
    "@types/node": "^22.0.0"
  }
}
```

Then create `tsconfig.json` at project root:

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "Node16",
    "moduleResolution": "Node16",
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```

Then run `npm install` to install all dependencies.

CRITICAL: `"type": "module"` is required. `"module": "Node16"` in tsconfig.json requires `.js` extensions on all relative imports in source files — this is enforced by TypeScript and will cause runtime failures if omitted.
